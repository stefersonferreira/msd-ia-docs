# Refinamento do Problema

**Estudo de Caso:** Sistema SMNC

Após criar o documento de visão inicial, refinamos a definição do problema para garantir que está focada e acionável.

## 1. Aprofundamento da Definição

Uma definição de problema bem refinada é essencial para guiar todas as decisões subsequentes do projeto.

::: info OBJETIVO
Refinar a definição do problema para torná-la mais específica, mensurável e alinhada com os objetivos de negócio.
:::

## 2. O Prompt de Refinamento

```md
Com base no documento de visão inicial do SMNC, refine a definição do problema seguindo a estrutura:

1. **Problema Atual**: Descreva o estado atual de forma específica e mensurável.
   - O que está acontecendo hoje?
   - Quais são os indicadores que comprovam o problema?
   - Qual o custo atual do problema?

2. **Problema Ideal**: Descreva o estado desejado de forma clara.
   - Como seria o cenário ideal?
   - Quais indicadores mediriam o sucesso?
   - Qual o valor esperado da solução?

3. **Gap de Problema**: Identifique a diferença entre atual e ideal.
   - O que precisa mudar?
   - Quais são as barreiras?
   - Quais são as oportunidades?

4. **Declaração Refinada**: Crie uma declaração de problema focada e acionável.
```

::: tip NOTA DO SÊNIOR
Uma definição de problema bem refinada evita que o projeto perca foco e garante que todas as soluções estejam alinhadas com o objetivo real.
:::

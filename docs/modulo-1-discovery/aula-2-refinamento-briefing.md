# Refinamento do Contexto

**Estudo de Caso:** Sistema SMNC (Monitoramento de Não Conformidades)

Após o briefing inicial, refinamos o contexto e aprofundamos os requisitos identificados.

## 1. Aprofundamento dos Requisitos

Nesta etapa, detalhamos os requisitos identificados no briefing inicial, garantindo que não haja ambiguidades.

::: info OBJETIVO
Refinar e detalhar os requisitos iniciais, transformando ideias vagas em especificações claras e acionáveis.
:::

## 2. O Prompt de Refinamento

```md
Com base no briefing inicial do SMNC, detalhe cada um dos seguintes pontos:

1. **Dores Identificadas**: Para cada uma das 5 dores, descreva:
   - Como ela se manifesta no dia a dia?
   - Qual o impacto financeiro/operacional?
   - Como o sistema resolve especificamente?

2. **Atores e Objetivos**: Para cada ator identificado, defina:
   - Quais são suas responsabilidades?
   - Quais informações ele precisa visualizar?
   - Quais ações ele precisa executar no sistema?

3. **Riscos**: Para cada risco da matriz, detalhe:
   - Probabilidade de ocorrência
   - Impacto se ocorrer
   - Mitigação proposta
```

::: tip NOTA DO SÊNIOR
O refinamento transforma requisitos de alto nível em especificações detalhadas que podem ser validadas com stakeholders e implementadas pelo time de desenvolvimento.
:::
